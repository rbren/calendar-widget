# 0.1.0

*Initial release — 2026-03-03*

## Features

- **Core calendar component** (`CalendarWidget`) with month grid view, header navigation, and day cells
- **Date selection** — single date and multi-date selection via `value` / `onChange`
- **Range selection** — `mode="range"` for two-click start/end date range picking with live hover preview
- **Date constraints** — `minDate`, `maxDate`, and `disabledDates` props to restrict selectable dates
- **Localization** — `locale` prop for `Intl.DateTimeFormat`-based month/year, weekday, and accessible day labels
- **Configurable week start** — `weekStartsOn` prop (0–6)
- **CSS custom properties** — 16 design tokens for colors (including range and preview colors), sizing, fonts, and border radius
- **Accessibility** — full keyboard navigation (arrow keys, Home/End, PageUp/PageDown), roving tabindex, descriptive `aria-label` on every day cell (e.g. "Saturday, March 15, 2026 (today, selected)"), ARIA roles (`grid`, `gridcell`, `columnheader`), `aria-selected`, `aria-disabled`, `aria-current="date"`, `aria-live="polite"` on month label
- **External value sync** — `useCalendarState` automatically syncs `viewDate` and `focusedDate` when the `value` prop changes externally
- **Public API exports** — `CalendarWidget`, `CalendarHeader`, `CalendarGrid`, `CalendarDayCell` components; `useCalendarState` hook; `getCalendarDays`, `isSameDay`, `isDateInRange`, `isDateDisabled`, `isDateRange`, `isDateBetween`, `formatMonthYear`, `formatDayLabel` utilities; `DateRange`, `CalendarWidgetProps`, `CalendarHeaderProps`, `CalendarGridProps`, `CalendarDayCellProps` type exports
- **Development server** — `npm run dev` starts a Vite dev server with a demo page

## Build & Tooling

- Vite library build producing ES module (`.mjs`) and CommonJS (`.cjs`) bundles
- TypeScript declaration files in `dist/types/`
- CSS output in `dist/style.css`
- `sideEffects: ["*.css"]` in `package.json` to prevent bundlers from dropping CSS imports
- ESLint + Prettier configuration
- Vitest testing infrastructure with jsdom environment and `@testing-library/react`
- GitHub Actions CI pipeline (lint, typecheck, test, build, npm audit)
- Dependabot configuration for automated dependency updates
- `engines` field requiring Node ≥ 20.19.0
