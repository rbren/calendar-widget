# 0.1.0

*Initial release — 2026-03-03*

## Features

- **Core calendar component** (`CalendarWidget`) with month grid view, header navigation, and day cells
- **Date selection** — single date and multi-date selection via `value` / `onChange`
- **Range selection** — `mode="range"` for two-click start/end date range picking with live hover preview
- **Quick navigation** — click the month/year heading to drill up into a month picker (4×3 grid), then a year picker (12-year range). `quickNavigation` prop (default `true`) to enable/disable
- **Today button** — `showTodayButton` prop (default `true`) renders a "Today" button below the header for quick return to the current month. `todayButtonLabel` prop for i18n
- **ISO week numbers** — `showWeekNumbers` prop adds a column displaying ISO 8601 week numbers
- **Custom day rendering** — `renderDay` prop accepts a function `(dayNumber, info) => ReactNode` to replace or augment day cell content
- **Lifecycle callbacks** — `onMonthChange(month)` fires when the displayed month changes via navigation; `onDayFocus(date)` fires when keyboard focus moves to a new day
- **Date constraints** — `minDate`, `maxDate`, and `disabledDates` props to restrict selectable dates
- **Localization** — `locale` prop for `Intl.DateTimeFormat`-based month/year, weekday, and accessible day labels
- **Configurable week start** — `weekStartsOn` prop (0–6)
- **CSS custom properties** — 16 design tokens for colors (including range and preview colors), sizing, fonts, and border radius
- **Accessibility** — full keyboard navigation (arrow keys, Home/End, PageUp/PageDown), roving tabindex, descriptive `aria-label` on every day cell (e.g. "Saturday, March 15, 2026 (today, selected)"), ARIA roles (`grid`, `gridcell`, `columnheader`), `aria-selected`, `aria-disabled`, `aria-current="date"`, `aria-live="polite"` on month label. Month and year picker grids have their own keyboard navigation and ARIA attributes. Nav button labels dynamically adapt to the active view
- **External value sync** — `useCalendarState` automatically syncs `viewDate` and `focusedDate` when the `value` prop changes externally
- **Public API exports** — `CalendarWidget`, `CalendarHeader`, `CalendarGrid`, `CalendarDayCell`, `MonthPicker`, `YearPicker` components; `useCalendarState` hook; `getCalendarDays`, `isSameDay`, `isDateInRange`, `isDateDisabled`, `isDateRange`, `isDateBetween`, `getISOWeekNumber`, `formatMonthYear`, `formatDayLabel` utilities; `DateRange`, `CalendarView`, `DayRenderInfo`, `CalendarWidgetProps`, `CalendarHeaderProps`, `CalendarGridProps`, `CalendarDayCellProps`, `MonthPickerProps`, `YearPickerProps` type exports
- **Development server** — `npm run dev` starts a Vite dev server with a demo page

- TypeScript declaration files in `dist/types/`

- Dependabot configuration for automated dependency updates
